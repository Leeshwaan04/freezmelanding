Follow these steps to take your matchmaking platform live on **freezme.in**.

## 1. Database Setup (Supabase)
- [ ] Create a project at [Supabase](https://supabase.com).
- [ ] Go to **SQL Editor** and run the following script to create your leads table:

```sql
CREATE TABLE applications (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  full_name TEXT NOT NULL,
  email TEXT NOT NULL,
  phone TEXT NOT NULL,
  dob DATE,
  gender TEXT,
  city TEXT,
  profession TEXT,
  preferences JSONB,
  about JSONB,
  submitted_at TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now()) NOT NULL
);

-- Enable RLS and public insert
ALTER TABLE applications ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Allow public submissions" ON applications FOR INSERT TO public WITH CHECK (true);
```

## 2. Code Deployment (Vercel)
- [ ] Push your code to a private **GitHub** repository.
- [ ] Import the repository into [Vercel](https://vercel.com).
- [ ] Configure the following **Environment Variables** in Vercel:
  - `NEXT_PUBLIC_SUPABASE_URL`: Your Supabase Project URL.
  - `NEXT_PUBLIC_SUPABASE_ANON_KEY`: Your Supabase Anon Key.

## 3. Domain Configuration (Hostinger)
- [ ] In Vercel Settings > **Domains**, add `freezme.in`.
- [ ] Go to your **Hostinger Panel** for `freezme.in`.
- [ ] Update DNS records as specified by Vercel:
  - **A Record**: Point `@` to Vercel's IP.
  - **CNAME**: Point `www` to `cname.vercel-dns.com`.

## 4. Final Polish & Analytics
- [ ] **Google Analytics**: Update the tracking ID (G-XXXXXXXXXX) in `src/app/layout.tsx`.
- [ ] **Test Submission**: Complete a test application on the live site to verify data appears in Supabase.
